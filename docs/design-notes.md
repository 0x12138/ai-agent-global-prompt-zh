# 设计说明

这份全局提示词的目标不是让模型“更像某个产品”，而是让不同 AI 编码助手在工程任务中表现得更稳定。

## 为什么使用中文

对主要使用中文和国内开发环境的用户，全局提示词使用中文可以减少语气、格式和本地化判断上的漂移。技术名词、命令、API 字段和错误原文保留英文，以保证精确性。

## 为什么强调最小改动

AI 编码助手常见问题不是“不会写”，而是改得太多、抽象太早、顺手重构、引入无关差异。全局提示词把“最小必要改动”和“保留项目风格”放在核心位置，是为了降低 diff 噪音和回归风险。

## 为什么单独强调 SQL 性能

SQL 和数据访问问题很容易在功能开发时被忽略，等线上数据量变大才暴露。提示词要求默认考虑查询范围、索引、分页、投影和 N+1 问题，是为了让性能约束前置到设计阶段。

## 为什么保留 `.wolf/`

`.wolf/` 是跨工具项目记忆层，适合多个 AI 编码工具共同处理一个源码项目时共享上下文。规则中特别限制它只用于源码项目，避免在纯文档、知识库或临时目录中产生无意义文件。

## 为什么保留 CodeGraph

CodeGraph 适合结构化代码理解，例如符号定义、调用关系和影响分析。提示词把它与 `rg` 分工：结构问题优先 CodeGraph，文本搜索优先 `rg`，避免重复扫描和低效探索。

## 可删减部分

如果你的工具或项目不使用以下能力，可以直接删除对应章节：

- `.wolf/`
- CodeGraph
- 网络搜索规则
- 个性化与记忆

如果你只需要最短版本，可以使用 [`templates/minimal.md`](../templates/minimal.md)。
